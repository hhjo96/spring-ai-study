## 2. RAG 옵션 개요

UI의 RAG 채팅 탭에서 선택할 수 있는 5가지 옵션입니다.

| RAG 옵션 | 설명 | 추가 파라미터 | 추천 용도 |
|---|---|---|---|
| QuestionAnswerAdvisor (basic) | 사용자 질문을 그대로 VectorStore에서 검색하여 답변 생성 | 없음 | 간단한 질문, 빠른 응답 |
| CompressionQueryTransformer (compression) | 대화 기록을 참고하여 질문을 압축/보완한 후 검색. ChatMemory와 함께 사용 | conversationId (대화 ID) | 다회차 대화, 문맥 유지 필요 시 |
| RewriteQueryTransformer (rewrite) | 사용자 질문을 LLM이 검색에 최적화된 형태로 재작성한 후 검색 | 없음 | 모호한 질문, 구어체 질문 |
| TranslationQueryTransformer (translation) | 질문을 대상 언어(한국어)로 번역한 후 검색. 영문 문서에 한국어 질문할 때 유용 | 없음 | 영문 문서 검색, 다국어 환경 |
| MultiQueryExpander (multiQuery) | 원본 질문을 3개의 다양한 질문으로 확장하여 각각 검색 후 결과를 합침 | 없음 | 불명확한 질문, 폭넓은 검색 필요 시 |

## 3. 설정 파라미터 가이드

| 항목 | 설명 |
|---|---|
| 유사도 점수 (score) | 0.0~1.0 사이 값. 높을수록 엄격한 필터링. 기본값 0.5 권장. 검색 결과가 없으면 0.3으로 낮춰보기. |
| 소스 필터 (source) | 특정 문서만 검색하고 싶을 때 파일명을 정확히 입력. 비워두면 전체 검색. |
| 대화 ID (conversationId) | Compression 모드에서만 사용. 같은 ID로 여러 질문하면 대화 문맥 유지. 다른 ID로 바꾸면 새 대화 시작. |
| 파일명 확인 | ETL 시 입력한 파일명이 source 메타데이터에 저장됨. 소스 필터에는 원본 파일명 그대로 입력. |

## 4. 옵션별 테스트 시나리오

### 4.1 QuestionAnswerAdvisor (기본)

가장 기본적인 RAG 방식으로, 질문을 그대로 VectorStore에서 검색합니다. 소스 필터와 유사도 점수 변경 효과를 함께 테스트합니다.

| # | 질문 | 소스 필터 | 점수 | 기대 결과 |
|---|---|---|---|---|
| 1 | 코스모스 테크놀로지의 유연근무제 코어타임은 몇 시부터 몇 시까지야? | sample-company-policy.txt | 0.5 | 오전 10시~오후 4시 |
| 2 | 연차 휴가 일수는 근속 연수에 따라 어떻게 달라져? | sample-company-policy.txt | 0.5 | 1년 미만 매월 1일, 1년 이상 15일, 3년 이상 추가 |
| 3 | 코스모스 AI 어시스턴트의 Professional 플랜 가격과 포함 기능은? | sample-product-guide.txt | 0.5 | 월 200만원, 50명, API 10만회 등 |
| 4 | Spring AI 개발 환경 구축 시 필요한 JDK 버전은? | sample-dev-setup.docx | 0.5 | 매뉴얼에 명시된 JDK 버전 응답 |
| 5 | 소스 필터 없이 "컨퍼런스 참석비 지원 횟수는?" 질문 | (비움) | 0.5 | 연간 2회 (사내 규정에서 검색) |
| 6 | 유사도 점수를 0.9로 올려서 질문해보기 | (비움) | 0.9 | 더 엄격한 필터링으로 결과 변화 확인 |

> **테스트 포인트:** #5번은 소스 필터 없이 전체 문서에서 검색되는지 확인. #6번은 점수를 0.9로 높여서 결과 변화를 관찰합니다.

---

### 4.2 CompressionQueryTransformer (압축)

대화 문맥을 유지하며 질문을 압축하는 방식입니다. 반드시 같은 대화 ID로 연속 질문하여 문맥 유지를 확인하세요.

| # | 질문 | 소스 필터 | 점수 | 기대 결과 |
|---|---|---|---|---|
| 1 | [첫 번째] 코스모스 테크놀로지의 초과근무 수당은 어떻게 계산돼? | (비움) | 0.5 | 통상임금 1.5배 (휴일 2배) |
| 2 | [두 번째, 같은 ID] 그럼 휴일에 근무하면 얼마야? | (비움) | 0.5 | "휴일근무"를 대화 문맥에서 이해하고 2배 응답 |
| 3 | [세 번째, 같은 ID] 월 한도는? | (비움) | 0.5 | "초과근무 월 한도"로 이해하고 52시간 응답 |
| 4 | [네 번째, 새 ID] 새 대화 ID로 "월 한도는?" 질문 | (비움) | 0.5 | 문맥 없이 응답 → 결과 비교 |

> **테스트 포인트:** #1~#3을 같은 클라이언트 ID(user-robbie-01)로 순서대로 질문하여 문맥 유지 확인. #4번은 다른 ID로 바꿔서 문맥 없는 응답과 비교합니다.

---

### 4.3 RewriteQueryTransformer (재작성)

구어체나 모호한 질문을 LLM이 검색에 최적화된 형태로 재작성합니다. 일상적인 표현으로 질문해보세요.

| # | 질문 | 소스 필터 | 점수 | 기대 결과 |
|---|---|---|---|---|
| 1 | 회사에서 머리 아프면 쉬는 거 몇 번 되는 거야? | (비움) | 0.5 | "병가"로 재작성되어 유급병가 30일 응답 |
| 2 | 그 AI 제품 쓰려면 컴퓨터가 어느 정도 좋아야 돼? | sample-product-guide.txt | 0.5 | 시스템 요구사항 응답 (CPU, RAM 등) |
| 3 | 승진하려면 얼마나 오래 다녀야 돼? | sample-company-policy.txt | 0.5 | 승진 연한 (사원→3년, 대리→4년 등) |
| 4 | 회사 컴퓨터에 USB 꽂아도 돼? | sample-company-policy.txt | 0.5 | 외부 USB 저장장치 사용 금지 응답 |

> **테스트 포인트:** 일상적인 구어체 질문이 공식적인 검색 용어로 변환되어 정확한 결과를 반환하는지 확인합니다.

---

### 4.4 TranslationQueryTransformer (번역)

질문을 한국어로 번역하여 검색합니다. 영문 문서(PDF, HTML)에 한국어로 질문하거나, 영문으로 질문하여 한국어 문서에서 검색되는지 테스트합니다.

| # | 질문 | 소스 필터 | 점수 | 기대 결과 |
|---|---|---|---|---|
| 1 | How to set up a development environment for Spring AI? | sample-dev-setup.docx | 0.5 | 한국어로 번역되어 검색 → 매뉴얼 기반 응답 |
| 2 | 자연어 처리의 주요 기술은 무엇인가요? | (비움) | 0.5 | 위키백과 NLP 문서에서 검색 |

> **테스트 포인트:** #1은 영문 질문이 한국어로 번역되어 검색되는지, #2번은 한국어 질문이 그대로 처리되는지 확인합니다.

---

### 4.5 MultiQueryExpander (다중 쿼리)

하나의 질문을 3개의 다양한 질문으로 확장하여 검색합니다. 폭넓은 주제로 질문하여 여러 문서에서 결과가 모이는지 확인하세요.

| # | 질문 | 소스 필터 | 점수 | 기대 결과 |
|---|---|---|---|---|
| 1 | 직원 복지 제도에 대해 알려줘 | (비움) | 0.5 | 식대, 교통비, 교육비, 건강검진 등 통합 응답 |
| 2 | AI 기술의 실무 활용 방법은? | (비움) | 0.3 | 제품 가이드 + 위키백과에서 폭넓게 검색 |
| 3 | Spring AI 프로젝트 시작 방법 | (비움) | 0.5 | PDF, 개발환경 매뉴얼, HTML 문서 등에서 검색 |
| 4 | 코스모스 테크놀로지 전반적인 회사 정보 | (비움) | 0.5 | 사내 규정 + 제품 가이드에서 복합 검색 |

> **테스트 포인트:** 다양한 문서에서 검색된 결과가 통합되어 포괄적인 답변이 생성되는지 확인합니다. 점수를 0.3으로 낮춰 더 많은 결과를 얻어보세요.
